# 自動収益プログラム計画

確認日: 2026-06-16

## 採用する初期戦略

最初は「無料ホスティング + アフィリエイト商品発見サイト」で始めます。

理由:

- 初期投資を0円にできる
- 商品在庫、決済、配送、返金対応を持たなくてよい
- 楽天Web Service APIで商品データを自動取得できる
- GitHub Actionsで毎日自動更新できる
- 不正クリック、SNSスパム、無断スクレイピングに寄らずに作れる

## 収益化ルート

### 1. 楽天アフィリエイト

このリポジトリの中心ルートです。

楽天Web ServiceのIchiba Item Search APIは2026-04-01版が公開されており、`applicationId` と `accessKey` を使って楽天市場の商品を検索できます。`affiliateId` を設定するとアフィリエイト導線を組み込めます。

参照: https://webservice.rakuten.co.jp/documentation/ichiba-item-search

### 2. A8.net

日本向けのASPとして追加候補です。公式サイトでは無料会員登録が案内されており、広告利用にも費用はかからないと説明されています。提携後、発行された広告リンクを記事内に手動追加します。

参照: https://www.a8.net/

### 3. Amazonアソシエイト

商品数が多く、世界展開にも向きます。ただし審査があり、Amazon公式ページでは180日以内の売上実績がない場合は申し込みが否認される旨が説明されています。最初から主軸にせず、サイト公開後に追加するのが安全です。

参照: https://affiliate.amazon.co.jp/

### 4. Google AdSense

アクセスが増えてからの広告候補です。Google公式ヘルプでは、独自コンテンツ、ポリシー準拠、18歳以上、サイトHTMLへのアクセスなどが要件として示されています。自動生成だけの薄いページでは審査に弱いので、まずは記事品質を上げます。

参照: https://support.google.com/adsense/answer/9724?hl=ja

## 初期投資

必須費用は0円です。

任意費用:

- 独自ドメイン: 年1,000円から3,000円程度
- 有料ホスティング: 月0円から1,000円程度
- 有料AI/API: 最初は不要

## 人間が必要な作業

完全自動化できない部分:

- 楽天、A8.net、Amazon、AdSenseなどのアカウント登録
- 本人確認、銀行口座、税務情報の入力
- 各サービスの審査対応
- APIキーやアフィリエイトIDの取得

プログラム側で自動化する部分:

- 商品検索
- スコアリング
- ページ生成
- サイトマップ生成
- 毎日更新
- 広告表記とプライバシー表記の設置

## 次にやること

1. GitHubにこのリポジトリを作ってpushする
2. GitHub Pagesの公開元を「GitHub Actions」にする
3. 楽天Web Serviceでアプリを作成する
4. GitHub Secretsに `RAKUTEN_APPLICATION_ID`、`RAKUTEN_ACCESS_KEY`、`RAKUTEN_AFFILIATE_ID` を設定する
5. `src/config.json` のテーマを1つに絞り、実体験ベースの記事文を増やす

## 禁止する運用

- 自分で広告をクリックする
- botでアクセスやクリックを水増しする
- 他サイト記事をコピーする
- SNSへ大量自動投稿する
- 医療、金融、法律など高リスク領域で根拠薄い記事を量産する

この方針なら、月1円を狙うための最小構成として安全寄りに始められます。
